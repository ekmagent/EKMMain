## Project Overview

MedicareYourself (medicareyourself.com) — a Medicare insurance broker website built on **Next.js 16 App Router** with TypeScript. The site has 192+ auto-generated service pages, educational content, state-specific plan guides, and a heavy SEO automation layer (50+ scripts).

## Commands

- **Dev server:** `npm run dev`
- **Build:** `npm run build` (runs `scripts/build-service-index.js` first, then `next build`)
- **Start:** `npm run start`
- **Lint:** `npx eslint .` (ESLint 9, flat config)
- **Build service index only:** `node scripts/build-service-index.js`

There are no test scripts configured.

## Architecture

### Routing & Content

- **App Router** — no Pages Router usage
- `/services/[slug]` — 192+ dynamically-discovered service pages. Each is a standalone `.tsx` file; `scripts/build-service-index.js` scans the directory and writes `service-index.json` (consumed by sitemap and the services index page)
- `/learn/[slug]` — educational guides (TSX)
- `/medicare-supplement/{state}` — state-specific Medigap pages (NJ, PA)
- `/medicare-faq/[slug]` — FAQ pages indexed via `faq-index.json`
- `/tools/*` — admin tools protected by cookie auth (`tools_auth` cookie, `TOOLS_PASSWORD` env var, middleware at root)
- `/api/contact` — contact form webhook; `/api/tools/auth` — admin login

### Key Data Flow

- **No database.** All content is static `.tsx` files committed to git
- **Medicare cost figures** live in `lib/medicare-figures.ts` — single source of truth for 2026 costs (Part A deductible, Part B premium, etc.). All pages import `MEDICARE` or `MF` from here. Update this file annually
- **`service-index.json`** and **`faq-index.json`** are build-time artifacts generated by scripts, consumed by `app/sitemap.ts` and index pages
- **Site URL** resolved in `lib/site.ts` via `NEXT_PUBLIC_SITE_URL` env var, falling back to Vercel URL or `medicareyourself.com`

### Components

Seven shared components in `/components`: `Header`, `Footer`, `QuoteWidget`, `FAQSection`, `PhoneCTA`, `ZipCTA`, `SchemaMarkup`. No UI component library — all custom Tailwind.

### Styling

- **Tailwind CSS 4** with PostCSS
- Brand colors defined as CSS variables in `globals.css`: primary teal `#2D6E5F`, CTA orange `#E8842A`, muted `#3A5048`, border `#D0E4DE`, section bg `#F6FAF8`
- Font: Google Montserrat

### Scripts (`/scripts`)

50+ Node.js scripts for SEO automation. Key categories:
- **Build/Index:** `build-service-index.js`, `build-compact-pages.js`, `build-faq-pages.js`
- **SEO Research:** `orchestrator.js` (quality audit), `seo-research.js`, `autoresearch-*.js`, `competitor-keyword-analysis.js`
- **Monitoring:** `citation-oracle.js`, `serp-tracker.js`, `fetch-gsc-data.js`, `fetch-ga4-ai-traffic.js`
- **Content Gen:** `generate-artifacts.js`, `humanize-content.js`, `expert-quotes.js`

These scripts require various API keys (Anthropic, Perplexity, Serper, Moz, Google service accounts).

### CI/CD

- GitHub Actions workflow at `.github/workflows/seo-research.yml`
- **Daily (2pm ET):** citation oracle, SERP tracker, quality checks
- **Weekly (Mondays):** full SEO pipeline — GSC data, keyword research, page generation, site audit
- Deployed on **Vercel**

### Config

- `next.config.ts` — MDX support (`.md`/`.mdx` page extensions), `/hub` → `/services` redirects
- `tsconfig.json` — `@/*` path alias to root, strict mode
- ESLint 9 flat config extending `next/core-web-vitals` + TypeScript

## Domain Rules

- Service pages follow a standardized structure: metadata + schema markup, breadcrumbs, hero with CTA, FAQ accordion, phone CTA, internal links
- Every page must have JSON-LD schema markup (use `SchemaMarkup` component)
- Never hardcode Medicare dollar amounts — always import from `lib/medicare-figures.ts`
